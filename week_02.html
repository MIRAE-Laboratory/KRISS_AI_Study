왜 마크다운으로 바꿀까요?

PDF 그대로 쓰지 않고 변환하는 이유

🤖 AI·에이전트 친화적
  • LLM이 텍스트+구조를 읽기 좋음. RAG, 요약, 챗봇 컨텍스트로 사용하기 편함
  • 포스터·리포트 재생성 시 "이 마크다운 기준으로 만들어줘"라고 지시하기 쉽음

  • PDF → 마크다운 한 번 해 두면, 포스터·PPT·요약·검색에 같은 소스를 재사용 가능
  • 버전 관리(예: Git)에도 텍스트라서 유리함
🔧 수정·재가공이 쉬움
  • 제목·목록·이미지 참조가 명확해 수동 수정이나 스크립트로 후처리하기 좋음

🛠️ 실습 1-1: PDF에서 텍스트 추출

페이지별로 텍스트만 가져오기

단계

1. 테스트용 PDF 하나 준비 (예: sample.pdf를 스터디 폴더에)

2. 아래와 비슷한 스크립트를 pdf_to_text.py 로 저장

3. 터미널에서 python pdf_to_text.py 실행 후 출력 확인

import fitz  # PyMuPDF

def extract_text(pdf_path: str) -> str:
    doc = fitz.open(pdf_path)
    text_parts = []
    for page in doc:
        text_parts.append(page.get_text())
    doc.close()
    return "\n\n".join(text_parts)

if __name__ == "__main__":
    path = "sample.pdf"  # 본인 PDF 경로로 변경
    text = extract_text(path)
    print(text[:1500])  # 앞 1500자만 확인
⚠️ 주의: 경로에 한글이 있으면 Windows에서 오류가 날 수 있으니, 영문 경로를 쓰거나 path = r"C:\경로\sample.pdf" 형태로 raw 문자열을 쓰세요.

🛠️ 실습 1-2: PDF에서 이미지 추출

내장된 그림을 폴더에 저장하기

목표: 각 페이지에서 이미지 리스트를 받아, 지정한 폴더에 page0_img0.png 형태로 저장

import fitz
import os

def extract_images(pdf_path: str, out_dir: str = "images"):
    os.makedirs(out_dir, exist_ok=True)
    doc = fitz.open(pdf_path)
    for page_num in range(len(doc)):
        page = doc[page_num]
        for img_index, img in enumerate(page.get_images()):
            xref = img[0]
            base = page.get_image_base64(xref)
            ext = "png"  # 또는 이미지 타입에 따라 jpg 등
            name = f"page{page_num}_img{img_index}.{ext}"
            path = os.path.join(out_dir, name)
            with open(path, "wb") as f:
                f.write(fitz.decode_image(base))
    doc.close()

Antigravity 워크플로란?

반복 작업을 에이전트에게 "한 번에" 맡기기

📌 워크플로의 의미
  • "이 폴더에 있는 여러 마크다운 파일을 각각 포스터(또는 HTML/이미지)로 만들어줘"처럼 여러 단계·여러 파일을 한 번에 지시하는 방식
  • Antigravity는 자연어로 단계를 설명하면, 에이전트가 스크립트를 짜고 실행까지 제안함
🎯 2주차에서 할 것
  • 실습 1에서 만든 markdown/ (또는 테스트용 md 몇 개)를 Antigravity에 폴더 경로로 알려 주기
  • "이 폴더 안의 각 .md 파일을 학술 포스터 형식의 HTML로 변환해 줘. 출력은 posters/ 에 저장해 줘." 같은 한 문장 지시로 워크플로 요청
  • 에이전트가 생성한 스크립트(반복문 + md 읽기 + 포스터 생성)를 검토하고 실행해 보기

🎯 실습 2: Antigravity로 md → 포스터 워크플로

자연어 한 문장으로 "폴더 전체 처리" 요청

Step 1. 준비

  • 실습 1에서 만든 markdown/ 폴더가 있다면 그대로 사용
  • 없다면: 수동으로 마크다운 파일 2~3개를 한 폴더(예: md_for_poster/)에 넣어 두기

Step 2. Antigravity에 넣을 지시문 예시

"markdown 폴더 안에 있는 각 .md 파일을 읽어서, 학술 포스터 형식의 HTML로 만들어 줘. 각 결과는 posters 폴더에 파일명과 같은 이름으로 저장해 줘. 포스터에는 제목, 개요, 본문 요약, 참고 문헌 섹션이 있으면 좋겠어."

Step 3. 확인할 것

  • 에이전트가 제안한 반복 처리 로직(어떤 폴더를 읽고, 어떤 방식으로 HTML을 만드는지)이 맞는지
  • 실행 시 경로(현재 프로젝트 기준 상대 경로)가 올바른지
  • posters/ 에 HTML 파일이 생성되었는지, 브라우저로 열어 보기
💡 Tip: "한 개의 md만 먼저 포스터로 만들어 보고, 그다음에 for 문으로 폴더 전체에 적용해 줘"처럼 단계를 나누어 지시해도 됩니다.

import streamlit as st

st.title("📄 PDF → 마크다운 변환기")
st.caption("2주차: KRISS Agentic AI 스터디")

folder = st.text_input("PDF가 있는 폴더 경로", placeholder="예: ./pdfs")

  • "마크다운 폴더 경로"를 추가 입력받아, "포스터 생성" 버튼으로 Antigravity가 만든 스크립트를 호출하거나, 간단한 md→HTML 변환을 앱 안에 넣기
  • 변환 중 진행률 표시 (st.progress)
💡 3주차부터는 이 앱에 챗봇을 붙일 예정이므로, 지금은 "PDF→md 변환"만 안정적으로 앱에 붙여 두면 충분합니다.

⚠️ 주의사항과 팁

실습할 때 꼭 생각하면 좋은 것들

🔒 저작권·개인정보
  • 변환하는 PDF가 본인 소유이거나 이용 허락이 있는지 확인
  • 연구실·기관 자료는 보안 정책에 맞게 사용
📂 경로
  • Windows: \ 대신 / 또는 raw 문자열 r"C:\경로" 사용 권장
  • Streamlit은 실행하는 터미널의 현재 디렉터리가 기준이므로, 상대 경로 사용 시 위치 확인
📦 큰 PDF·많은 파일
  • 한 번에 수백 개 PDF를 돌리면 메모리·시간이 많이 듦. 작은 폴더로 먼저 테스트
  • 스캔 PDF(이미지만 있는 문서)는 OCR 없이 텍스트가 안 나옴 — 나중에 Tesseract 등 OCR 파이프라인 추가 가능

2주차 정리

오늘 기억할 세 가지

1️⃣ PDF → 마크다운
  • PyMuPDF로 텍스트·이미지 추출 후, 페이지 순으로 이어서 마크다운 파일로 저장
  • 폴더 단위 일괄 변환 스크립트까지 작성해 두면 재사용이 편함
2️⃣ Antigravity 워크플로
  • "이 폴더의 md들을 각각 포스터로 만들어줘"처럼 한 문장 지시로 반복 작업 위임
  • 에이전트가 제안한 스크립트·경로를 꼭 검토한 뒤 실행
3️⃣ 앱으로 구현
  • Streamlit으로 PDF 폴더 경로 입력 + 변환 실행 + 결과 안내까지 연결
  • 다음 주부터는 이 앱에 챗봇·페르소나를 붙여 갑니다
