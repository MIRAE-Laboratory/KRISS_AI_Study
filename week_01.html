Agentic AI 활용 트렌드

도구 사용에서 에이전트 지휘로 — 패러다임 전환

📈 현재 트렌드
  • AI는 이제 "한 번에 한 질문"이 아닌 목표 지향 자동 실행
  • 검색·코드·파일·리포트까지 파이프라인을 에이전트가 수행
  • 연구·기업 현장에서 Agent-first 플랫폼과 툴이 빠르게 확산
🔬 연구개발 맥락
  • 논문 조사 → 요약 → 초안 작성, 데이터 분석 → 시각화 → 보고서 등
  • 사람은 목표를 주고 결과를 검증하는 역할로 이동

Tool vs Agent — 핵심 차이

이 구분을 이해하면 과정의 절반이다

구분AI 도구(Tool)AI 에이전트(Agent)
비유전동 드릴숙련된 목수
동작사용자가 버튼을 눌러야만 동작목표를 주면 스스로 계획·실행
예시"이 문장 번역해줘""이 논문 읽고 한국어 요약 보고서 만들어줘"
실패 시에러 메시지 출력원인 분석 후 다른 방법 시도
맥락매번 새로 시작이전 작업 맥락을 기억하며 진행
도구 사용없음(그 자체가 도구)검색, 코드 실행, 파일 처리 등 활용

연구개발(R&D)에 활용할 전략

에이전트를 어떻게 설계·운영할 것인가

🧱 오케스트레이션
  • 에이전트·도구를 레고 블록처럼 조합
  • "어떤 에이전트에게 어떤 역할을 줄까?"를 설계
  • 코딩보다 아키텍처 사고가 중요
⚖️ 책임
  • 에이전트 출력에 대한 최종 책임은 사람에게
  • 환각·편향을 검증하고 수정하는 단계 필수
  • "AI가 그렇게 말했다"는 변명이 되지 않음
🕵️ Human-in-the-Loop (HITL)
  • 반복 작업은 자동화, 중요 판단 지점에서는 사람이 개입
  • 해석·윤리·창의적 방향 결정은 AI에 위임하지 않음

Human-in-the-Loop (HITL)

"어디서 사람이 개입할 것인가?" — 설계의 핵심 질문

에이전트 시작
➔
결과 생성
➔
⚠️ 사람 검토
➔
계속 진행 또는 수정 지시
➔
최종 결과
💡 HITL이 필요한 순간
  • 데이터 해석·통계적 판단
  • 윤리적 판단이 필요한 경우
  • 연구 가설·창의적 방향 선택
  • 대외 공개·논문·보고서 최종 확정 전
⚠️ HITL 없을 때 위험
  • 존재하지 않는 논문 인용(환각)
  • 편향된 데이터로 잘못된 결론
  • 저작권·정확성·신뢰도 문제

🛠️ Antigravity — 에이전트 우선 개발 환경

Google의 에이전트 기반 IDE

📌 Antigravity란?
  • 에이전트가 코드 작성·실행·수정까지 수행하는 에이전트 퍼스트 플랫폼
  • 자연어 지시 → 계획 → 코드 생성 → 실행 → 결과 확인 흐름 지원
🔗 활용
  • 연구 포스터, 스크립트, 시각화 등 지시만으로 결과물 생성
  • 8주 과정에서 PDF→마크다운, 워크플로, 앱 구현 등에 단계적으로 활용

🎯 실습: Antigravity로 연구포스터 만들기

자연어로 지시해 포스터 초안 생성

"내 연구 주제(또는 관심 주제)를 한 문장으로 알려주면, 학술 포스터 형식의 마크다운/HTML 초안을 만들어줘."

진행 순서

1. Antigravity에서 새 채팅(또는 에이전트 세션) 시작

2. 위와 같은 지시를 자연스럽게 입력

3. 에이전트가 생성한 포스터 초안 검토

4. 제목, 섹션(배경, 방법, 결과, 결론 등), 레이아웃 수정 지시로 다듬기

1주차 정리

오늘 기억할 세 가지

1️⃣ 트렌드와 패러다임
  • AI "도구 사용" → AI "에이전트 지휘"
  • 실행은 에이전트, 목표 설정과 검증은 사람
2️⃣ 에이전트 ≠ 챗봇
  • 스스로 계획·도구 사용·수정 시도
  • "검색기"가 아니라 자율 연구 파트너에 가깝다
3️⃣ 연구개발 전략
  • 오케스트레이션, 책임, Human-in-the-Loop
  • Antigravity로 연구포스터까지 만들어 보며 첫 걸음 완료
